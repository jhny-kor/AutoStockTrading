# 구현 로드맵

## 1단계: 저장소 골격 만들기

먼저 공통 구조부터 고정하는 것을 권장합니다.

```text
src/
  brokers/
    base.py
    kis/
    ibkr/
  strategies/
  services/
  config/
  backtest/
tests/
scripts/
```

핵심 원칙은 `브로커`, `전략`, `운영 서비스`를 분리하는 것입니다.

## 2단계: 브로커 추상화

브로커마다 API는 달라도 인터페이스는 비슷하게 맞추는 것이 좋습니다.

예시 메서드:

- `get_quote(symbol)`
- `get_balance()`
- `get_positions()`
- `place_order(...)`
- `cancel_order(order_id)`
- `stream_ticks(symbols)`

이 인터페이스를 먼저 만들면 한국투자증권과 IBKR를 같은 전략 엔진에서 다룰 수 있습니다.

## 3단계: 모의투자 우선 연결

실전 계좌보다 아래 순서로 진행하는 것이 안전합니다.

1. 인증 토큰 발급
2. 시세 조회
3. 잔고 조회
4. 모의 주문
5. 주문 상태 추적
6. 예외 처리 및 재시도

이 단계에서는 수익보다 `정확한 상태 관리`가 더 중요합니다.

## 4단계: 전략 엔진 분리

전략 코드는 브로커 세부 구현을 몰라도 작동해야 합니다.

예시 책임:

- 매수/매도 신호 계산
- 종목 필터링
- 주문 수량 산정
- 손절/익절 규칙 판단
- 장 시작 전/장 종료 후 루틴

## 5단계: 운영 안전장치

자동매매에서는 이 부분이 필수입니다.

- 일별 최대 손실 한도
- 종목별 최대 비중 제한
- 연속 주문 실패 차단
- 시장 상태 점검
- 중복 주문 방지 키
- 모든 주문 이벤트 감사 로그 저장

## 6단계: 추천 MVP 범위

첫 버전은 아래 정도가 적당합니다.

- 한국주식 1개 브로커 연동
- 미국주식 1개 브로커 연동
- 단일 전략 1개
- 장중 시세 조회 + 시장가/지정가 주문
- 디스코드 또는 텔레그램 알림
- 주문/체결/에러 로그 저장

## 7단계: 문서화 우선순위

코드를 쓰기 시작하면 아래 문서도 함께 유지하는 것이 좋습니다.

- `docs/api-keys.md`: 키 발급, 권한, IP 등록 절차
- `docs/env.md`: 환경변수 설명
- `docs/strategy-spec.md`: 전략 규칙
- `docs/runbook.md`: 장애 대응과 수동 복구 절차
